UCFitness Dashboard

Fitbit APIを活用した、グループ対抗フィットネス・トラッキング・プラットフォーム

Visit Live Site →

Project Overview

UCFitness は、Fitbitユーザーが自身の活動データ（歩数、心拍数など）を可視化し、グループ間で競争できるダッシュボードアプリケーションです。標準のFitbitアプリでは難しい「特定グループ内でのランキング」や「長期間のデータ推移のカスタム比較」を実現するために開発されました。

Tech Stack

  • Frontend: Next.js (App Router), React, TailwindCSS, Recharts
  • Backend: Supabase (Database, Auth), Next.js API Routes
  • Integration: Fitbit Web API (OAuth 2.0)
  • Infrastructure: Vercel (Hosting), Cloudflare

Key Features

1. グループランキングシステム

ユーザーは特定のグループに参加し、歩数や活動距離を競うことができます。Supabaseのリアルタイム機能を活用し、ランキングは定期的に更新されます。

2. セキュアなOAuth認証

NextAuth.js を使用せず、Fitbit APIとの直接的なOAuth 2.0連携を実装。アクセストークンとリフレッシュトークンの管理をサーバーサイド（HttpOnly Cookie）で厳格に行い、セキュリティを担保しています。

3. データ可視化

Rechartsライブラリを使用し、日次・週次・月次の活動データをインタラクティブなグラフで表示。APIのレート制限（Rate Limits）を考慮し、データのキャッシング戦略も実装しています。

Challenges & Solutions

課題: Fitbit APIのレート制限（1時間あたり150リクエスト）が厳しく、多人数グループでの一斉更新時に制限に達する問題がありました。

解決策: データの取得をサーバーサイドでバッチ処理化し、結果をSupabaseにキャッシュすることで、Fitbit APIへの直接的なアクセスを最小限に抑えるアーキテクチャを採用しました。また、フロントエンド側でもSWRを活用して不要な再フェッチを防いでいます。

Future Roadmap

今後は、Apple Health や Google Fit との連携も視野に入れ、より多くのデバイスユーザーが参加できるプラットフォームへと進化させる予定です。
