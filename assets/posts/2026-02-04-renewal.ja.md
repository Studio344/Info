# ポートフォリオサイトを全面リニューアルしました

## リニューアルの背景

Studio344のポートフォリオサイトを全面的にリニューアルしました。旧サイトは縦長の1ページ構成で、プロジェクトへのリンクを並べただけのシンプルな構成でした。しかし、開発の知見を蓄積・発信する場として不十分であり、以下の課題を抱えていました。

- **コンテンツの薄さ**: プロジェクト名とリンクのみで、技術的な詳細がなかった
- **情報発信の場がない**: 開発ログやTipsを書く場所がなかった
- **拡張性の低さ**: 単一HTMLファイルにすべてを詰め込む構成で、メンテナンスが困難
- **必須ページの不足**: プライバシーポリシーやお問い合わせページがなかった

「単なるリンク集」から「技術的な発信ができるポートフォリオ」への進化を目指して、ゼロからの再設計に踏み切りました。

## 技術選定

リニューアルにあたって、以下の技術スタックを選定しました。

### ホスティング: GitHub Pages

静的サイトとして運用するため、**GitHub Pages** を選択しました。GitHubリポジトリに直接プッシュするだけでデプロイが完了するシンプルさが魅力です。独自ドメイン `studio344.net` も設定し、HTTPS対応も無料で利用できています。

### アニメーション: GSAP + ScrollTrigger

ページに動きを持たせるために **GSAP（GreenSock Animation Platform）v3.12.2** を採用しました。ScrollTriggerプラグインと組み合わせることで、スクロールに連動したフェードインやスライドアニメーションを実現しています。

GSAPを選んだ理由は以下の通りです：
- CSS Animationだけでは制御が難しい複雑なタイムラインが組める
- パフォーマンスが優秀（GPUアクセラレーション対応）
- 豊富なイージング関数で自然な動きを表現できる

### 多言語対応: i18next

日本語・英語の二言語対応には **i18next v23.2.3** を使用しています。HTMLの `data-i18n` 属性を利用した宣言的な翻訳指定により、JavaScriptコードと翻訳データを明確に分離できています。詳細は別記事で詳しく解説予定です。

### UI デザイン: Bento Grid + Aurora UI

デザインコンセプトとして **Bento Grid レイアウト** を採用しました。Apple風のカードベースUIにより、情報を視覚的に整理しつつ、モダンな印象を与えています。背景には **Aurora UI**（グラデーションオーブ + グリッドオーバーレイ + ノイズテクスチャ）を組み合わせ、奥行きのある視覚効果を実現しました。

## 主な変更点

### マルチページ構成への移行

旧サイトの単一ページから、目的別にページを分割しました。

| ページ | 内容 |
|--------|------|
| **index.html** | トップページ（ヒーロー + 自己紹介 + ナビゲーション） |
| **projects.html** | プロジェクト一覧 |
| **blog.html** | 開発ログ・技術記事 |
| **about.html** | 自己紹介・スキル・哲学 |
| **contact.html** | お問い合わせ |

### プロジェクト詳細ページ

各プロジェクトに専用の詳細ページを作成しました。iframeによるライブプレビュー、使用技術の一覧、パフォーマンスメトリクスなど、技術的な背景が伝わるコンテンツを充実させています。

### ブログ機能

Markdownファイルベースのシンプルなブログシステムを構築しました。`marked.js` でMarkdownをHTMLに変換し、`list.json` で記事の管理を行うフラットファイル方式です。CMSを導入せず、GitHubリポジトリで記事を管理できるため、開発者にとって自然なワークフローを維持しています。

### 必須ページの追加

- プライバシーポリシー
- 利用規約
- お問い合わせページ
- サイトマップ（`sitemap.xml`）

## デザインのこだわり

### レスポンシブ対応

CSS Grid と Flexbox を組み合わせ、デスクトップからモバイルまでシームレスに表示されるレイアウトを実装しました。ブレークポイントは600pxを基準にし、カードの配置やフォントサイズを段階的に調整しています。

### パフォーマンス最適化

- 外部ライブラリはCDN経由で読み込み、初回表示を高速化
- 画像ファイルは最小限に抑え、アイコンはSVGで統一
- アニメーションはGPUアクセラレーションを活用

## 今後の展望

リニューアルはまだ始まりに過ぎません。今後は以下の改善を予定しています。

- 技術記事の定期的な投稿（月2〜3本を目標）
- プロジェクトの追加（新規開発中のアプリを公開予定）
- ダークモード / ライトモードの切り替え機能
- ページ遷移アニメーションの強化

開発の記録を残すことで、自分自身の成長のログにもなりますし、同じ課題に直面している方の参考にもなれば嬉しいです。今後も不定期に更新していきますので、ぜひご覧ください。
