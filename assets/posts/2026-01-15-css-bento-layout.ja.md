# CSSグリッドで作る「Bento Layout」の設計パターン

## Bento Layoutとは

Apple が WWDC23 で採用したことで注目を集めた「Bento Layout（弁当箱レイアウト）」は、異なるサイズのカードをグリッド上に美しく配置するデザインパターンです。Studio344 のポートフォリオサイトでもこのレイアウトを採用しており、本記事ではその設計と実装のポイントを解説します。

## なぜ Bento Layout を選んだのか

ポートフォリオサイトを設計する際、以下の要件がありました。

- **情報密度**: 限られたスペースにAbout、Projects、Blogなどの異なる種類の情報を配置したい
- **視覚的な階層**: 重要な情報を大きく、補助情報をコンパクトに表示したい
- **レスポンシブ対応**: モバイルからデスクトップまで自然にレイアウトが変化してほしい

従来のカラムレイアウトでは単調になりがちですが、Bento Layout なら各セクションの重要度に応じてサイズを変えられるため、自然な視線の流れを作れます。

## CSS Grid による実装

### 基本構造

```css
.bento-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(300px, 1fr));
  gap: 1.2rem;
  max-width: 900px;
  margin: 0 auto;
  padding: 2rem;
}
```

`auto-fit` と `minmax()` の組み合わせが重要です。これにより、画面幅に応じて自動的にカラム数が変化します。

### カードのスパン制御

特定のカードを2カラム分の幅に広げるには、以下のように設定します。

```css
.bento-card.wide {
  grid-column: span 2;
}

@media (max-width: 768px) {
  .bento-card.wide {
    grid-column: span 1; /* モバイルでは1カラムに */
  }
}
```

### ガラスモーフィズムとの組み合わせ

各カードに `backdrop-filter` を使ったガラスモーフィズム効果を加えることで、奥行きのある表現が可能です。

```css
.bento-card {
  background: rgba(255, 255, 255, 0.03);
  border: 1px solid rgba(255, 255, 255, 0.06);
  border-radius: 24px;
  backdrop-filter: blur(24px);
  padding: 2rem;
}
```

## 設計で注意したポイント

### 1. コンテンツファースト

レイアウトありきではなく、まず「何を伝えたいか」を明確にしてから、それに合うグリッド配置を決めました。ヒーローセクションは全幅、Tech Stack は中サイズ、ナビパネルはコンパクトに、という具合です。

### 2. 余白の統一

`gap: 1.2rem` をグリッド全体で統一し、カード内の `padding` も統一しています。これにより「弁当箱」の仕切りのような整然とした印象を保てます。

### 3. アクセシビリティ

カードのフォーカス状態を明示し、キーボードナビゲーションでスムーズに移動できるようにしています。`tabindex` と `:focus-visible` の組み合わせが有効です。

## モバイル対応の工夫

Bento Layout の最大の課題はモバイル対応です。デスクトップでは美しいグリッドも、スマートフォンでは縦に並ぶだけになりがちです。

Studio344 では以下のアプローチで解決しています。

- **768px以下**: すべてのカードを1カラムに配置
- **カード順序の最適化**: `order` プロパティで、モバイルではヒーロー → About → Projects → Blog の順に再配置
- **タッチ対応**: ホバーエフェクトの代わりにタップフィードバックを追加

## まとめ

Bento Layout は「見せ方」と「情報設計」を両立できる優れたパターンです。CSS Grid の `auto-fit` と `minmax()` を活用すれば、レスポンシブ対応も比較的容易に実現できます。ポートフォリオサイトやダッシュボードなど、異なる種類の情報を一画面に配置したい場面で特に効果を発揮します。
