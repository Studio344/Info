### 1. 🔴 ローディングステート：コンテンツのJSレンダリング時にスケルトンUIが無い

**📍 問題箇所:** `script.js` L28-30, L163-210 / `blog.js` L108-120  
**🔍 問題の詳細:**  
Featured Projects、Latest Blog、ブログ記事一覧がすべてJSによる非同期描画だが、`fetch` 完了まで空の `<div>` が表示される。ネットワーク遅延時にユーザーは何も見えない白い（黒い）空間を体験する。ブログ記事読み込みは「読み込み中...」テキストのみ。

**💡 提案:**
- **スケルトンUI** を各セクションに実装（CSS-only のパルスアニメーション付きプレースホルダー）
- Featured Projects: カードの形をしたスケルトン × 2
- Latest Blog: 行状のスケルトン × 3
- Blog記事本文: ヘッダー + 段落のスケルトン
- ブログ一覧: グリッドカードのスケルトン

**📊 期待効果:** 知覚パフォーマンスが大幅向上。CLS (Cumulative Layout Shift) の改善。  
**⏱️ 工数:** M（中）

### 2. 🔴 fetch エラー時のエラーステートが不十分

**📍 問題箇所:** `script.js` L22（`.catch(() => { statPosts.textContent = "-"; })`）、`blog.js` L200-203  
**🔍 問題の詳細:**  
- `projects.json` の fetch 失敗時、プロジェクトカードが一切表示されず空のまま（エラーメッセージなし）
- ブログ一覧 fetch 失敗時も空のグリッドが放置される
- ホームの Latest Blog セクションは `.catch(() => {})` で完全に握りつぶし
- Stat カウンターの "-" 表示は何が起きたのかユーザーに伝えない

**💡 提案:**
- 各 fetch 失敗時に **リトライボタン付きのフレンドリーなエラーカード** を表示
- 「コンテンツを読み込めませんでした。再読み込みしてください」+ リトライアイコン
- i18n 対応（ja/en 両方のエラーメッセージ）

**📊 期待効果:** ユーザーの離脱防止、トラブルシュートの手助け  
**⏱️ 工数:** S（小）

### 3. 🔴 モバイルでのヒーロー CTA ボタンの視認性とタップ領域

**📍 問題箇所:** `styles.css` L1699-1704  
**🔍 問題の詳細:**  
モバイルで CTA ボタンが `width: 200px` 固定＋ `flex-direction: column` になるが、Secondary ボタン（Read Blog）がゴースト系スタイルのため背景と溶け込みやすい。また、ボタン間のギャップが `0.7rem` と狭く、誤タップリスクがある。

**💡 提案:**
- Secondary ボタンの border を `rgba(255,255,255,0.25)` に強化
- ボタン間のギャップを `1rem` に拡大
- モバイルでのボタン幅を `min(240px, 80vw)` に変更し画面サイズに柔軟対応
- タップフィードバックとして `:active` 時の `scale(0.96)` を追加（現在 `0.98` で微弱）

**📊 期待効果:** CTA のコンバージョン率向上、モバイルのタッチ精度改善  
**⏱️ 工数:** S（小）

### 4. 🔴 ハンバーガーメニューのアニメーションとフォーカストラップ不在

**📍 問題箇所:** `styles.css` L1506-1576 / ハンバーガー JS（ui.js に未実装 → 別途 inline か）  
**🔍 問題の詳細:**  
- メニュー展開時、ハンバーガーアイコンが `×` に変わるアニメーションが無い（SVGの入れ替え処理自体が見当たらない）
- フルスクリーンメニュー展開後の **フォーカストラップ** が実装されていない。Tab キーでメニュー外の要素にフォーカスが移動可能
- メニュー内リンクのスタガードアニメーション（順番に出現）が無い
- `Escape` キーでメニューを閉じる処理の有無が不明

**💡 提案:**
1. ハンバーガー → ×（クロス）のCSS transition アニメーション実装
2. フォーカストラップ: `focusin` イベントで menu 内の要素のみにフォーカスを制限
3. `Escape` キーでメニュー閉じ + フォーカスをハンバーガーボタンに戻す
4. メニューリンクにスタガードフェードイン（`transition-delay` 0.05s ずつ）

**📊 期待効果:** WCAG 2.1 AA 準拠（モーダルダイアログのフォーカス管理）、UX 品質向上  
**⏱️ 工数:** M（中）

### 6. 🟡 ページ遷移時のビジュアルフィードバック不足

**📍 問題箇所:** `<meta name="view-transition" content="same-origin" />` は設定済み  
**🔍 問題の詳細:**  
View Transitions API のメタタグは設置されているが、カスタムの遷移アニメーション CSS（`::view-transition-old`, `::view-transition-new`）が `animation-duration` 設定のみで、実際のカスタムアニメーション（フェード、スライド等）が定義されていない。結果、ブラウザデフォルトのクロスフェードのみ。

**💡 提案:**
- ページ出入り時のカスタム View Transition アニメーション定義
- ヒーローカード → `view-transition-name: hero` で持続する要素アニメーション
- ロゴ → `view-transition-name: logo` でシームレスなブランド要素遷移

**📊 期待効果:** SPA ライクな体験、プレミアム感向上  
**⏱️ 工数:** M（中）

### 16. 🟡 カード内のフォーカス順序とキーボードナビゲーション

**📍 問題箇所:** `index.html` ホームナビカード  
**🔍 問題の詳細:**  
- `home-nav-card--connect` は `<div>` 要素で、キーボードでフォーカスできない。中の SNS リンクには個別にフォーカスが当たるが、カード全体としてのインタラクションが不一致

- ブログカードが `<div>` ベースの場合は `tabindex="0"` + `role="article"` を付与

**📊 期待効果:** WCAG 2.1 AA キーボードナビゲーション準拠  
**⏱️ 工数:** S（小）

- ヘッダーに月/太陽アイコンのトグル追加
- ただし、Aurora 背景やグロー効果はダークテーマに最適化されているため、ライトテーマでは別のビジュアル処理が必要

**📊 期待効果:** ユーザーの好みに対応、アクセシビリティ向上  
**⏱️ 工数:** L（大）  
**⚠️ 注意:** ダークテーマの完成度が高いため、ライト対応のROIは要検討

### 23. 🟢 印刷スタイルシートの追加

**📍 問題箇所:** `styles.css` に `@media print` が無い  
**🔍 問題の詳細:**  
ブログ記事やAboutページを印刷/PDF保存する際、Aurora 背景やグロー効果がそのまま出力され、読みにくいまたはインクを浪費する。

**💡 提案:**
```css
@media print {
  .aurora-container, .grid-overlay, .noise-overlay, 
  .spotlight-overlay, .header, .back-to-top, .site-footer { display: none; }
  body { background: #fff; color: #000; }
  .bento-card { background: #fff; border: 1px solid #ddd; box-shadow: none; }
}
```

**📊 期待効果:** 印刷/PDF 出力の品質向上  
**⏱️ 工数:** S（小）

### 24. 🟢 ブログ記事の目次（TOC）のスクロールスパイ

**📍 問題箇所:** `blog.js` TOC 関連処理  
**🔍 問題の詳細:**  
目次は折りたたみ対応だが、現在読んでいるセクションのハイライト（スクロールスパイ）が無い。

**💡 提案:**

- 対応する TOC リンクに `.toc-active` クラスを付与
- `color: var(--accent-lavender)` + 左ボーダー でアクティブ表示

**📊 期待効果:** 長文記事でのナビゲーション改善  
**⏱️ 工数:** M（中）

### 25. 🟢 CSS の `word-break: break-all` の修正

**📍 問題箇所:** `styles.css` L3883 （`:lang(ja)` セクション）  
**🔍 問題の詳細:**  
`:lang(ja)` に `word-break: break-all` が設定されているが、これは日本語テキストには不要（日本語は元々任意の位置で改行可能）。逆に、日本語テキスト内の英単語が途中で折り返される問題が発生する可能性がある。

**💡 提案:**
- `word-break: break-all` → `word-break: normal` に変更
- `overflow-wrap: break-word` は維持（長いURLの折り返しに有効）
- 必要に応じて `word-break: keep-all` を検討（韓国語対応がある場合）

**📊 期待効果:** テキスト組版の品質向上、英単語の不自然な分割防止  
**⏱️ 工数:** S（小）
